Explanation of Categories

What does this method do?

Model-based methods create a mathematical model that represents how a time series should behave under normal conditions. If a new data point significantly deviates from the model's prediction, it is flagged as an anomaly.

How does it work?

The method builds a predictive model using historical data and calculates expected values for new observations. If a point falls outside the expected range, it is considered an anomaly.

Simple Example

Imagine you check your electricity bill every month, and it's always around $50-$60. Suddenly, one month you receive a $500 bill - this is an anomaly.

Real-World Example

In industrial monitoring, machines have an expected operating temperature range. If a sensor detects a sudden and significant increase in temperature, the system identifies it as an anomaly, possibly indicating a malfunction.

How to use this method?

  • Train a model using past data.
  • Predict future values based on historical trends.
  • Compare actual values to predictions.
  • Flag significant deviations as anomalies.

What does this method do?

Density-based methods detect anomalies by examining how closely grouped data points are. Normal points cluster together, while anomalies appear isolated.

How does it work?

A density-based algorithm calculates the local density around each point. If a point is far from all clusters, it is considered an anomaly.

Simple Example

Imagine a concert where most people stand in groups. If one person is standing alone far away, they would seem unusual.

Real-World Example

Credit card fraud detection: most transactions occur in expected locations and within a common price range. If a transaction is made from a different country and for an unusually high amount, it might be flagged as fraudulent.

How to use this method?

  • Gather data - collect time series values.
  • Apply a density-based algorithm - e.g., DBSCAN (Density-Based Spatial Clustering of Applications with Noise).
  • Flag outliers - points in low-density areas are considered anomalies.

What does this method do?

Histogramming divides the data into bins and analyzes the frequency of values within each bin. Rare values fall into sparsely populated bins, marking them as potential anomalies.

How does it work?

The algorithm creates a histogram of historical data and then checks whether new values fit within common bins or belong to rare ones.

Simple Example

If you count how many times students attend school per month, and most students attend 18-22 days, but one student attends only one day, this would stand out.

Real-World Example

In monitoring website response times, if most pages load within 200ms but one request takes 5000ms, this suggests an issue with the server.

How to use this method?

  • Build a histogram - define bins based on past data.
  • Compare new values - if a point falls into an unexpected bin, it is flagged as an anomaly.

What does this method do?

Discord detection identifies the most unusual subsequences in a time series - those that significantly differ from all others.

How does it work?

The method compares each subsequence with all others and finds the one that deviates the most.

Simple Example

If you usually exercise for 30 minutes daily but one day suddenly exercise for 3 hours, that would be unusual.

Real-World Example

In medical ECG monitoring, most heartbeats follow a predictable pattern. If a patient suddenly exhibits an irregular pattern, it could indicate a health problem.

How to use this method?

  • Extract all subsequences from the time series.
  • Measure their similarity using distance metrics.
  • Flag the most dissimilar subsequence as a potential anomaly.

What does this method do?

Dissimilarity-based methods compare different subsequences within a time series and detect patterns that are significantly different from the rest.

How does it work?

These methods identify sequences that do not match known patterns by measuring similarity between subsequences.

Simple Example

If you sign your name the same way every day, but one day your signature looks entirely different, it would be an anomaly.

Real-World Example

In manufacturing, machines follow predictable operational patterns. If a machine's vibration pattern changes significantly, it could indicate wear or an impending breakdown.

How to use this method?

  • Extract subsequences from the data.
  • Measure dissimilarity using distance metrics.
  • Identify highly dissimilar sequences as anomalies.

What does this method do?

This method analyzes periodic patterns in a time series. If the expected frequency suddenly changes or disappears, it may indicate an anomaly.

How does it work?

It applies a Fourier Transform or other frequency analysis techniques to examine the dominant frequencies in the data.

Simple Example

A clock rings every hour. If it suddenly rings every 30 minutes or stops ringing entirely, something is wrong.

Real-World Example

In power grid monitoring, electricity should operate at a stable 50Hz frequency. A deviation from this frequency might indicate a fault in the power system.

How to use this method?

  • Analyze periodic signals in the data.
  • Compare observed frequencies with expected ones.
  • Flag sudden frequency changes as anomalies.

What does this method do?

Information theory-based methods measure changes in entropy (randomness). A sudden increase in entropy suggests the system has become less predictable, indicating an anomaly.

How does it work?

It calculates entropy or compression complexity over time and detects periods where the randomness significantly increases.

Simple Example

If a classroom is usually quiet but suddenly students start yelling chaotically, the unpredictability has increased, signaling a shift.

Real-World Example

In cybersecurity, network traffic usually follows predictable patterns. If entropy suddenly increases, it may indicate a cyberattack or an unexpected system behavior.

How to use this method?

  • Compute entropy over time.
  • Monitor for unexpected increases.
  • Flag high-entropy periods as potential anomalies.
